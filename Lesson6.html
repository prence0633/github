Introduction - Data Types

Today, the Teacher guides Frankie and John in learning about Data Types and how to design an Entity-Relationship Diagram (ERD). Each attribute in a table must have a clear Data Type: Text for words, Number for numeric values, Date for dates, and Boolean for true/false flags. Choosing the correct Data Type prevents errors and ensures calculations or comparisons work properly. By assigning the correct types and linking tables with Primary Keys and Foreign Keys, they can create an ERD that clearly shows entities, attributes, and the relationships between them. Through this, Frankie and John can organize information efficiently and visualize how real-world data connects in a database system.

Glossary — Key Terms

Text
String of characters; used for names, descriptions.
Number
Numeric type for counts, IDs, quantities.
Date
Stores calendar dates; allows date arithmetic.
Boolean
True/False values; useful for flags like IsClaimed.
ERD
Entity-Relationship Diagram — visual map of entities and relationships.
