Summary — Basic Database Structure

Table: Organizes data into rows (records) and columns (fields) for easier storage and retrieval.

Row (Record): Represents a single entry in a table, such as one student's information.

Column (Field): Represents a specific attribute or category of data, such as Name, Section, or Student_ID.

Keys (Primary and Foreign): Ensure data accuracy and connection between tables — the Primary Key uniquely identifies each record, while the Foreign Key links related data across tables.

Glossary — Key Terms

Database
A structured collection of related data stored so it can be easily accessed and managed.
Table
Organizes data in rows (records) and columns (fields).
Row (Record)
Represents a single entry or instance in a table — for example, one student’s information.
Column (Field)
Represents a specific attribute or category of data — for example, Name, Section, or Student_ID.
Primary Key
A column (or set of columns) that uniquely identifies each record in a table.
Foreign Key
A column that stores primary key values from another table to create a relationship between them.
