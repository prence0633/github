Introduction — Table Relationships

Today, John investigates his lost items and discovers that they are connected to other students through their records. Frankie introduces Table Relationships to reveal the connections between tables. Relationships organize data in different ways: One-to-One links each item to a single record (like Student → ID Card), One-to-Many links one record to multiple related entries (like Student → Attendance Records), and Many-to-Many connects multiple records to multiple entries (like Students → Clubs). By understanding these relationships, John can efficiently track items, see connections across tables, and organize information clearly.

Summary — Table Relationships

One-to-One: Each row in Table A matches exactly one row in Table B (e.g., Student ↔ ID Card).

One-to-Many: One row in Table A relates to multiple rows in Table B (e.g., Student ↔ Attendance records).

Many-to-Many: Multiple rows in Table A relate to multiple rows in Table B (e.g., Students ↔ Clubs).

Glossary — Key Terms

One-to-One
One row in Table A corresponds to one row in Table B.
One-to-Many
One row in Table A corresponds to many rows in Table B.
Many-to-Many
Many rows in Table A relate to many rows in Table B, usually via a join table.
